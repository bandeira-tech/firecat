The Problem

The internet has a landlord problem.

Today's infrastructure is concentrated in the hands of a few companies. The people who create value have the least control over it.

Infrastructure concentration

A handful of cloud providers control where data lives, what it costs to store, and who can access it. Small builders pay rent to incumbents.

Adversarial advertising

Marketing budgets flow to platform owners. Advertisers pay to interrupt users. Users get a degraded experience. The platforms pocket the spread.

Data hostage dynamics

When an app dies, users lose their data. When a platform changes terms, developers lose their audience. No portability. No sovereignty.

The Network

A protocol for the public network.

Firecat is the canonical protocol built on B3nd. If B3nd is the foundation, Firecat is the building — open to everyone, owned by the people inside it.

Community-owned infrastructure

Firecat defines the schemas, authentication model, and URI conventions for a public, community-owned data network. Nodes are operated by people like you. Not a company. A network.

Every byte on Firecat is owned by its creator, addressable by any application, and encrypted by default. The data layer inherits everything from B3nd — user ownership, client-side encryption, URI-based addressing.

Economics

Economics designed for flow, not accumulation.

Three layers hold Firecat together. No layer extracts from the others. Every participant is compensated for what they contribute.

⚡

Write gas

Every write costs a predictable fraction of a cent, paid as FCAT. Each message funds its own relay. Reads are free.

⚖

Operator staking

Node operators stake FCAT to join the network. Staked operators receive a share of accumulated write fees and relay rewards.

⇄

Relay rewards

When a node relays a message to a peer, it earns a relay credit. Bilateral attestation between peers creates verifiable proofs.

No tokens are burned. All fees circulate back into the economy. The network's health is measured by throughput — active users, messages per second — not by token price.

Token

FCAT: gas, not speculation.

A utility token with UTXO-based gas semantics. Its value comes from network activity, not from hype cycles.

FCAT

One token, four functions

FCAT powers the entire Firecat economy. Spent to write, staked to operate, earned by relaying, used for governance. When the network is used, the token has purpose.

firecat quick start 
import { HttpClient } from "@bandeira-tech/b3nd-web";

const client = new HttpClient({
  url: "https://testnet-evergreen.fire.cat"
});

// Write to the public network
await client.receive([
  "mutable://open/my-app/hello",
  { message: "it works" }
]);

// Read it back
const result = await client.read(
  "mutable://open/my-app/hello"
);
console.log(result.record?.data);
// { message: "it works" }

Anti-whale by design

Rewards are weighted by the square root of your stake. One whale cannot outrun a thousand participants. The math enforces what policy cannot.

Operator A stakes
100 FCAT
Operator A earns
9.1% of rewards
Operator B stakes
10,000 FCAT
Operator B earns
90.9% of rewards

B has 100x the stake but only 10x the reward. sqrt(100)=10, sqrt(10000)=100. The math is simple; the implications aren't.

Community

Start for free. Stay because it's fair.

The barrier to entry is zero. The network grows when anyone can join — not just those who can afford to.

Faucet

Sponsors

Applications and advertisers can pre-fund gas for their users. Meta-transactions mean users never see a fee.

Existing staked accounts can vouch for new users, granting initial gas from a subsidy pool. Bring your friends.
